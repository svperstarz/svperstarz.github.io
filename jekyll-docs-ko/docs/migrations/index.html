블로그 마이그레이션

만약 다른 블로그 시스템을 Jekyll 로 바꾸는 경우라면, Jekyll 의 마이그레이션 도구가 도움이 될 것입니다. 이 페이지에 소개된 방법들 중 대다수가 기존 시스템 데이터베이스에 읽기 권한이 있어야지만 Jekyll 포스트를 생성할 수 있습니다. 기존 시스템의 엔트리를 기반으로 하여 _posts 디렉토리에 .markdown 포스트를 생성합니다.

마이그레이션 준비하기

마이그레이션 도구는 자체적으로 다른 의존요소들을 많이 가지고 있어서, jekyll-import 라는 별개의 Gem 으로 제공됩니다. 단순히 이 Gem 을 설치하는 것 만으로 모든 준비가 끝나며, Jekyll 의 기본 명령어 인터페이스를 통해서 사용할 수 있습니다.

$ gem install jekyll-import --pre

아래 설명된 마이그레이션 도구를 실행할 준비가 모두 끝났습니다. 막히는 부분이 있다면, 해당 마이그레이션 도구에 대한 도움말을 확인하세요.

$ jekyll help import           # => 마이그레이션 도구 목록 보기
$ jekyll help import IMPORTER  # => 마이그레이션 도구 IMPORTER 의 도움말 보기

여기서 IMPORTER 는 특정 마이그레이션 도구의 이름입니다.

Note: 마이그레이션 결과물을 다시 한번 확인하세요

마이그레이션 도구가 게시된 포스트와 비공개 포스트를 구별하지 못할 수도 있습니다. 따라서, Jekyll 의 컨텐츠가 자신이 의도한대로 생성되었는지 확인하세요.

WordPress

WordPress 내보내기 파일

만약 hpricot 이 설치되어 있지 않다면, gem install hpricot 을 실행해야 합니다. 그 다음, WordPress 내보내기 유틸리티를 사용해서 블로그 자료를 내보내기 합니다. 내보내기 파일을 wordpress.xml 로 저장했다고 가정햇을 때, 실행 명령어는 다음과 같습니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/wordpressdotcom";
    JekyllImport::WordpressDotCom.process({ :source => "wordpress.xml" })'
ProTip™: WordPress.com 내보내기 도구

WordPress.com 계정으로부터 마이그레이션 하는 경우, 다음 URL: `https://YOUR-USER-NAME.wordpress.com/wp-admin/export.php` 에서 내보내기 도구를 사용할 수 있습니다.

WordPress MySQL 서버 연결 사용하기

WordPress MySQL 서버에 직접 연결하여 데이터를 가져오는 방법은 다음과 같습니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/wordpress";
    JekyllImport::WordPress.process({:dbname => "database", :user => "user", :pass => "pass"})'

만약 Webfaction 을 사용하고 있고 SSH tunnel 을 설정해야 한다면, 반드시 호스트명을 (127.0.0.1) 로 지정하세요. MySQL 의 인증 시스템에서는 127.0.0.1 과 localhost 가 동일하지 않아 연결이 거절될 수 있습니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/wordpress";
    JekyllImport::WordPress.process({:host => "127.0.0.1", :dbname => "database", :user => "user", :pass => "pass"})'

또 다른 WordPress 마이그레이션 방법

위 방법도 잘 동작하지만, 일반적으로 WordPress 포스트와 페이지에 저장되어 있는 메타데이터 대부분을 잃게 됩니다. 페이지나 태그, 사용자 필드, 첨부 이미지 등을 내보내기 해야된다면, 다음 자료들이 유용할 것입니다:

  • Exitwp 은 파이썬으로 작성되었으며, 설정을 변경할 수 있고, 가능한 많은 메타데이터를 유지한 상태로, 하나 이상의 WordPress 블로그를 Jekyll (마크다운) 포맷으로 마이그레이션 하는 도구입니다.
  • A great article 에 WordPress 의 블로그 구조와 메타데이터 대부분을 그대로 유지하면서 Jekyll 로 마이그레이션 하는 방법이 각 단계별로 설명되어 있습니다.
  • wpXml2Jekyll 는 WordPress XML 파일을 읽어 마크다운 포스트를 생성하는 윈도우용 응용 프로그램입니다.

Drupal

Drupal 로부터 마이그레이션 한다면, 각 버전별로 사용할 수 있는 마이그레이션 도구가 따로 있습니다: - Drupal 6 - Drupal 7

$ ruby -rubygems -e 'require "jekyll/jekyll-import/drupal6";
    JekyllImport::Drupal6.process("dbname", "user", "pass")'
# ... or ...
$ ruby -rubygems -e 'require "jekyll/jekyll-import/drupal7";
    JekyllImport::Drupal7.process("dbname", "user", "pass")'

다른 호스트에 접속한다거나 테이블 접두어를 설정해야 한다면, Drupal 마이그레이션 도구 실행 명령 끝 부분에 필요한 파라메터를 입력합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/drupal6";
    JekyllImport::Drupal6.process("dbname", "user", "pass", "host", "table_prefix")'
# ... or ...
$ ruby -rubygems -e 'require "jekyll/jekyll-import/drupal7";
    JekyllImport::Drupal7.process("dbname", "user", "pass", "host", "table_prefix")'

Movable Type

Movable Type 을 마이그레이션하려면 다음과 같이 실행합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/mt";
    JekyllImport::MT.process("database", "user", "pass")'

Typo

Typo 를 마이그레이션하려면 다음과 같이 실행합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/typo";
    JekyllImport::Typo.process("database", "user", "pass")'

이 코드는 Typo 버전 4+ 에서만 테스트 되었습니다.

TextPattern

TextPattern 을 마이그레이션하려면 다음과 같이 실행합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/textpattern";
    JekyllImport::TextPattern.process("database_name", "username", "password", "hostname")'

위 명령은 _import 폴더의 상위 디렉토리에서 실행해야 합니다. 예를 들어, _import 가 /path/source/_import 에 있다면, /path/source 에서 명령을 실행합니다. 호스트명의 기본값은 localhost 이고 다른 변수들은 직접 지정해야 합니다. 특정 엔트리를 골라내기 위해서는 명령을 약간 수정할 필요가 있습니다. 위와 똑같이 실행할 경우, live 또는 sticky 상태의 모든 엔트리를 가져오게 될 것입니다.

Mephisto

Mephisto 를 마이그레이션하려면 다음과 같이 실행합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/mephisto";
    JekyllImport::Mephisto.process("database", "user", "password")'

데이터가 PostgreSQL 에 저장되어 있다면, 다음과 같이 실행합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/mephisto";
    JekyllImport::Mephisto.postgres({:database => "database", :username=>"username", :password =>"password"})'

Blogger (Blogspot)

Blogger 를 마이그레이션하려면, Blogger 에서 Jekyll 로의 마이그레이션 방법 을 참고하세요. 설명대로 잘 되지 않는다면, 아래 내용들을 참고해보세요:

  • @kennym 도 위 게시물의 내용대로 되지 않아서 간단한 마이그레이션 스크립트를 만들었습니다.
  • @ngauthier 는 RSS 피드가 아닌 Blogger 의 아카이브를 통해서 댓글까지 가져오는 다른 마이그레이션 도구를 만들었습니다.
  • @juniorz 는 Octopress 에 사용할 수 있는 또 다른 마이그레이션 도구를 만들었습니다. @ngauthier 가 만든 것과 비슷하지만, 포스트와 초안을 구분하고 태그와 고유주소도 가져올 수 있습니다.

Posterous

Posterous 기본 블로그를 마이그레이션 하려면, 다음과 같이 실행합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/posterous";
    JekyllImport::Posterous.process("my_email", "my_pass")'

계정 내의 다른 Posterous 블로그에 대해서는, 블로그의 blog_id 를 지정해야 합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/posterous";
    JekyllImport::Posterous.process("my_email", "my_pass", "blog_id")'

또, 고유주소를 유지하고 이미지도 가져오려고 시도하는 다른 Posterous 마이그레이션 도구 도 있습니다.

Tumblr

Tumblr 에서 마이그레이션 하려면 다음과 같이 실행합니다:

$ ruby -rubygems -e 'require "jekyll/jekyll-import/tumblr";
    JekyllImport::Tumblr.process(url, format, grab_images, add_highlights, rewrite_urls)'
# url    - String: 블로그 URL
# format - String: 생성되는 파일의 확장자. HTML 에서 Markdown 으로 변환되게
#          하려면 "md" 를 사용하세요. 기본값: "html".
# grab_images    - Boolean: 이미지 다운로드 여부. 기본값: false.
# add_highlights - Boolean: Liquid "highlight" 태그에서 코드 블록
#                  (공백문자 4 개로 들여쓰기 된 부분) 을 wrap 할지 여부.
#                  기본값: false.
# rewrite_urls   - Boolean: 기존 Tumblr 경로를 Jekyll 경로로 바꿀지 여부.
#                  기본값: false.
