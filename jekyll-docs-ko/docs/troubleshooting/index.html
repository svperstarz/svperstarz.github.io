문제해결

여기, Jekyll 설치나 사용에 문제가 생겼을 때 도움이 될 만한 몇 가지 팁이 있습니다. 아래 설명된 내용으로도 해결이 안되는 문제가 있다면, Jekyll 커뮤니티에서 조치할 수 있도록 이슈를 등록해 주세요.

설치 시 문제점

만약 Gem 설치 시 에러가 발생한다면, 루비 1.9.1 컴파일 확장기능 모듈의 헤더 파일이 필요한 것일 수 있습니다. 우분투 또는 데비안에서는 다음 명령을 실행합니다:

sudo apt-get install ruby1.9.1-dev

레드햇이나 CentOS, 페도라에서는 다음과 같이 실행합니다:

sudo yum install ruby-devel

NearlyFreeSpeech 에서는 명령 실행 전에 다음 환경변수를 설정하세요:

RB_USER_INSTALL=true gem install jekyll

OSX 에서는, RubyGems 를 업데이트 하세요:

sudo gem update --system

여전히 문제가 해결되지 않는다면, XCode 의 명령행 도구를 설치해야하는 것일 수 있습니다. 이 도구를 설치한 뒤, 다음 명령으로 Native Gem 을 설치합니다:

sudo gem install jekyll

Gentoo 에 RubyGems 를 설치하려면 다음 명령을 실행합니다:

sudo emerge -av dev-ruby/rubygems

윈도우즈에서는 RubyInstaller DevKit 을 설치해야 합니다.

Jekyll 실행 시 문제점

데비안이나 우분투에서는, Jekyll 실행파일을 터미널에서 사용하려면 $PATH 에 /var/lib/gems/1.8/bin/ 을 추가해야 할 수도 있습니다.

Base-URL 문제점

다음과 같이 base-url 옵션을 사용할 경우:

jekyll serve --baseurl '/blog'

… 아래 주소로 접속할 수 있는지 확인하세요:

http://localhost:4000/blog/index.html

아래 주소로는 접근할 수 없을 것입니다:

http://localhost:4000/blog

환경설정 문제점

환경설정의 우선순위는 다음과 같습니다:

  1. 명령행 플래그
  2. 환경설정 파일
  3. 기본값

다시 말해: _config.yml 에 설정된 옵션이 기본값 대신 사용되고, 명령행에 사용된 플래그들이 다른 모든 설정들보다 우선순위가 높습니다.

마크업 문제점

다양한 마크업 엔진들이 Jekyll 에 사용되고 있으며, 각 엔진만의 문제점이 있을 수 있습니다. 동일한 문제를 겪고 있는 이들에게 도움이 될 만한 정보를 이 페이지에 모아두었습니다.

Maruku

링크에 이스케이프해야 할 문자가 포함되어 있다면, 다음 문법을 사용하세요:

![Alt text](http://yuml.me/diagram/class/[Project]->[Task])

Maruku 는 <script src="js.js"></script> 와 같은 비어있는 태그를 <script src="js.js" /> 로 변환시킵니다. 이것은 Firefox 에서 문제를 일으키며 다른 브라우저에서도 문제가 될 가능성이 있습니다. 또, XHTML 관례에 위반됩니다. 간단한 해결방법으로는 여는 태그와 닫는 태그 사이에 공백을 넣는 것입니다.

RedCloth

버전 4.1.1 이후 버전이 notextile 태그를 준수하지 않습니다. 이것은 알려진 버그이며, 다행히도 4.2 버전에서 수정될 것입니다. 계속해서 4.1.9 버전을 사용할 수도 있지만, 테스트 환경은 4.1.0 을 필요로 합니다. 만약 notextile 태그를 지원하지 않는 버전의 RedCloth 를 사용할 경우, Pygments 의 구문 강조 기능이 정상적으로 동작하지 않을 수도 있습니다. 이런 경우에는 4.1.0 을 설치하는게 낫습니다.

Liquid

최신 버전인 2.0 에서는, 템플릿 안에서 {{ 가 고장난 것 같습니다. 이전 버전과는 다르게, 2.0 에서 {{ 를 사용하면 다음과 같은 에러가 발생합니다:

'{{' was not properly terminated with regexp: /\}\}/  (Liquid::SyntaxError)

발췌

v1.0.0 이후부터, Jekyll 에 자동으로 생성된 포스트 발췌 기능이 있었습니다. v1.1.0 이후부터는, Jekyll 이 이 발췌를 Liquid 에 전달하게 되어 레퍼런스가 존재하지 않는다거나 태그가 올바르게 닫히지 않았다는 이상한 에러를 발생시킵니다. 이런 문제가 발생하면, _config.yml 에 excerpt_separator: "" 설정을 추가하거나 이상한 문자열로 설정해보세요.

문제를 발견하면 알려주세요!

만약 버그를 발견했다면, GitHub 에 이슈를 생성해서 문제 상황과 대처법을 설명해주세요. 다른 이들이 볼 수 있도록 이 문서에 포함하겠습니다.
