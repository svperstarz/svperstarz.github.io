디렉토리 구조

Jekyll 의 핵심 역할은 텍스트 변환 엔진입니다. 시스템의 배경 컨셉은 다음과 같습니다. 사용자가 즐겨 사용하는 마크업 언어 (Markdown 이나 Textile 또는 일반 HTML) 로 작성된 문서를 레이아웃으로 포장합니다. 이 과정에서 사이트 URL 이 보여지는 방식이나 데이터가 레이아웃에 배치되는 방식, 그 밖에 많은 것들을 원하는대로 바꿀 수 있습니다. 최종적으로 정적 웹 사이트가 만들어지며, 텍스트 파일을 수정하는 것만으로 이 모든 일들이 가능합니다.

다음은 Jekyll 사이트의 일반적인 형태입니다:

.
├── _config.yml
├── _drafts
|   ├── begin-with-the-crazy-ideas.textile
|   └── on-simplicity-in-technology.markdown
├── _includes
|   ├── footer.html
|   └── header.html
├── _layouts
|   ├── default.html
|   └── post.html
├── _posts
|   ├── 2007-10-29-why-every-programmer-should-play-nethack.textile
|   └── 2009-04-26-barcamp-boston-4-roundup.textile
├── _site
└── index.html

각 항목이 하는 일은 다음과 같습니다:

파일 /
디렉토리		 설명

_config.yml

환경설정 정보를 보관합니다. 옵션은 대부분 명령어에 직접 지정하여 실행할 수도 있지만, 그렇게 따로 외워두는 것보다 이 파일에 저장하는 편이 더 쉽습니다.

_drafts

아직 게시되지 않은 초안들입니다. 파일명 형식에 날짜가 없습니다: title.MARKUP. 초안 사용하기를 읽어보세요.

_includes

필요에 따라 포스트나 레이아웃에 짜 맞출 수 있는, 재사용하기 좋은 파일들입니다: Liquid 태그 {% include file.ext %} 를 사용하면 _includes/file.ext 의 내용이 삽입됩니다.

_layouts

포스트를 감싸는 템플릿들입니다. 레이아웃은 다음 섹션에 소개되는 YAML 머리말에 의해 각 포스트 별로 선택됩니다. Liquid 태그 {{ content }} 를 사용하여 웹 페이지에 컨텐츠를 주입합니다.

_posts

말하자면 동적인, 자신의 컨텐츠입니다. 파일명 형식은 반드시 YEAR-MONTH-DAY-title.MARKUP 이어야 하며, 아주 중요합니다. 고유주소는 따로 수정이 가능하지만, 날짜와 마크업 언어 종류는 오로지 파일명에 의해서 결정됩니다.

_site

Jekyll 이 변환을 마친 후, 생성된 사이트가 저장되는 위치입니다 (기본값). .gitignore 파일에 이 디렉토리를 추가하는 것이 좋습니다.

index.html 및 다른 HTML, Markdown, Textile 파일들

파일에 YAML 머리말 섹션이 있다면 Jekyll 에 의해 변환됩니다. 사이트의 최상위 디렉토리나 위에 언급되지 않은 디렉토리 안의 모든 .html 이나 .markdown, .md, .textile 파일도 해당됩니다.

다른 파일/폴더

그 밖에 css 나 images 폴더, favicon.ico 파일 등 위에 언급되지 않은 다른 디렉토리와 파일들은 생성된 사이트에 그대로 복사됩니다. 다른 사이트들은 어떤 식으로 구성되었는지 궁금하다면, 이미 Jekyll 을 사용하는 사이트들이 많이 있으니 확인해보세요.
