변수

Jekyll 은 작업 대상 파일을 찾기 위해 사이트 내부를 검색합니다. YAML 머리말을 가진 파일이 작업 대상입니다. Jekyll 은 각각의 파일마다 Liquid 템플릿 시스템을 통해 다양한 데이터를 만들어냅니다. 사용가능한 데이터 목록은 다음과 같습니다.

전역 변수

변수 설명

site

사이트 정보 + _config.yml 의 환경설정 정보. 더 자세한 내용은 아래 참조.

page

페이지 정보 + YAML 머리말. YAML 머리말에 설정된 사용자 변수도 포함됩니다. 더 자세한 내용은 아래 참조.

content

레이아웃에 포함된 포스트 또는 페이지 컨텐츠. 포스트나 페이지 파일에는 정의되어 있지 않습니다.

paginator

이 변수는 환경설정 옵션 paginate 이 설정되어 있을 때 사용가능합니다. 더 자세한 내용은 Pagination 를 참조.

사이트 변수

변수 설명

site.time

현재 시간 (jekyll 명령을 실행한 시간).

site.pages

모든 페이지 목록.

site.posts

시간 역순의 모든 포스트 목록.

site.related_posts

포스트인 경우에, 최대 10 개의 관련 포스트가 이 변수에 담겨 있습니다. 기본적으로는, 처리속도가 빠른 대신 결과의 질이 낮은 방식을 사용합니다. 느린 속도를 감수하고 좋은 결과를 얻으려면, jekyll 명령을 --lsi (latent semantic indexing) 옵션과 함께 실행합니다.

site.categories.CATEGORY

CATEGORY 카테고리에 속한 모든 포스트 목록.

site.tags.TAG

TAG 태그가 붙은 모든 포스트 목록.

site.[CONFIGURATION_DATA]

명령행이나 _config.yml 에 설정된 모든 변수를 site 변수를 통해 사용할 수 있습니다. 예를 들어, 환경설정 파일에 url: http://mysite.com 가 있다면, 포스트나 페이지에서 site.url 를 사용할 수 있습니다. Jekyll 은 watch 모드에서도 _config.yml 파일의 변경사항을 자동으로 적용하지 않는다. 변수의 변경사항을 적용하려면 Jekyll 을 재시작해야 한다.

페이지 변수

변수 설명

page.content

페이지의 (처리되지 않은) 실제 컨텐츠.

page.title

페이지의 제목.

page.excerpt

페이지의 (처리되지 않은) 실제 미리보기 부분.

page.url

도메인을 제외한, 포스트 URL. 슬래쉬로 시작함. 예시, /2008/12/14/my-post.html

page.date

포스트에 할당된 날짜. 포스트의 머리말에 YYYY-MM-DD HH:MM:SS (UTC 기준), 또는 YYYY-MM-DD HH:MM:SS +/-TTTT (UTC 오프셋을 사용한 타임 존 지정. 예시, 2008-12-14 10:30:00 +0900) 형식으로 새로운 날짜/시간을 지정하여 덮어쓸 수 있다.

page.id

포스트에 대한 유일한 식별자 (RSS 피드에 유용함). 예시, /2008/12/14/my-post

page.categories

포스트가 속한 카테고리 목록. 카테고리는 _posts 디렉토리의 상위 디렉토리 구조로부터 파생된다. 예를 들어, /work/code/_posts/2008-12-24-closures.md 에는 이 변수가 ['work', 'code'] 로 설정되어 있다. 이 변수는 YAML 머리말 에서도 설정할 수 있다.

page.tags

포스트에 붙어있는 태그 목록. 이 변수는 YAML 머리말 에서도 설정할 수 있다.

page.path

페이지 또는 포스트의 실제 경로. 활용 예시: 페이지 또는 포스트의 GitHub 소스 링크. 이 변수는 YAML 머리말 에서도 설정할 수 있다.

ProTip™: 사용자 머리말을 사용하세요

모든 사용자 머리말은 page 로 사용할 수 있습니다. 예를 들어, 페이지 머리말에 custom_css: true 를 설정한 경우, page.custom_css 로 해당 값을 사용할 수 있습니다.

Paginator

변수 설명

paginator.per_page

페이지 당 포스트 수.

paginator.posts

페이지에서 사용할 수 있는 포스트들.

paginator.total_posts

전체 포스트 수.

paginator.total_pages

전체 페이지 수.

paginator.page

현재 페이지 번호.

paginator.previous_page

이전 페이지 번호.

paginator.previous_page_path

이전 페이지 경로.

paginator.next_page

다음 페이지 번호.

paginator.next_page_path

다음 페이지 경로.
Paginator 변수 사용 위치

이 변수들은 오직 인덱스 파일에서만 사용할 수 있지만, /blog/index.html 같이 하위 디렉토리에도 해당됩니다.
